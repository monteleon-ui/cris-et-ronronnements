# Contexte : Conformité RGAA pour cris-et-ronronnements

## 🎯 Objectif RGAA
Le site **cris-et-ronronnements** doit être conforme au **RGAA (Référentiel Général d'Amélioration de l'Accessibilité) version 4.1**, niveau **AA**.

---

## 📋 Checklist RGAA (mise à jour après optimisations)

### ✅ Critères déjà implémentés
- **Contrastes des couleurs** : Vérifiés via les classes Bootstrap et les variables CSS (ex: `--primary-color: #6f42c1` a un contraste de 7.4:1 avec le blanc).
- **Balises ARIA** : Utilisées dans les templates (ex: `role="banner"`, `role="main"`, `role="contentinfo"`, `aria-label`).
- **Structure sémantique** : Utilisation de balises HTML5 (`<header>`, `<main>`, `<footer>`, `<section>`, `<nav>`).
- **Alternatives textuelles** : Attributs `alt` pour les images (favicon, apple-touch-icon).
- **Navigation au clavier** : Bootstrap garantit une navigation accessible par défaut.
- **Skip to main content** : Lien de navigation pour les lecteurs d'écran (ajouté dans `main.ejs`).

### 🟡 Critères à vérifier
- **Images** : Toutes les images doivent avoir un attribut `alt` **descriptif** (même pour les icônes).
- **Formulaires** : Chaque champ doit avoir une étiquette (`<label>`) ou un attribut `aria-label`.
- **Liens** : Les liens doivent avoir un texte **clair et descriptif** (éviter "cliquez ici").
- **Multimédia** : Si des vidéos/audios sont ajoutés, fournir des sous-titres ou transcriptions.
- **Erreurs** : Les messages d'erreur (ex: 404, 500) doivent être **accessibles** et compréhensibles.

### ❌ Critères non applicables
- **Contenu dynamique** : Pas de contenu mis à jour dynamiquement sans rechargement de page (sauf si JavaScript est activé).
- **Audio/Vidéo en direct** : Non présent sur le site.

## 🛠️ Optimisations RGAA récentes (Juillet 2025)

### Améliorations implémentées
1. **Skip to main content** :
   - Ajout d'un lien invisible (accessible au clavier) dans `main.ejs`.
   - Permet aux utilisateurs de lecteurs d'écran de sauter directement au contenu principal.
   ```html
   <a href="#main-content" class="visually-hidden-focusable" style="position: absolute; left: -9999px; ...">
       Aller au contenu principal
   </a>
   ```

2. **Aria-label optimisés** :
   - Suppression des `aria-label` redondants (ex: `<a href="/">Accueil</a>` au lieu de `<a href="/" aria-label="Accueil - Page d'accueil">Accueil</a>`).
   - Conservation des `aria-label` utiles (ex: liens externes, icônes).

3. **Structure sémantique renforcée** :
   - Ajout de `id="main-content"` sur la balise `<main>`.
   - Ajout de `aria-labelledby` sur les sections.

4. **Pages d'erreur accessibles** :
   - Ajout de `role="main"` et `aria-labelledby` sur les pages 404 et 500.
   - Ajout de liens de navigation clairs.

5. **Contrastes vérifiés** :
   - `--primary-color: #6f42c1` sur blanc : **7.4:1** (AAA).
   - `--secondary-color: #0d6efd` sur blanc : **4.6:1** (AA).
   - Texte sur fond clair : **15.3:1** (AAA).

## 🔍 Outils pour vérifier la conformité RGAA

### Outils automatisés
| Outil | Commande/Installation | Lien |
|-------|------------------------|------|
| **axe-cli** | `npm install -g axe-cli` puis `axe http://localhost:3000` | [axe-core](https://github.com/dequelabs/axe-core) |
| **Lighthouse** | Intégré dans Chrome DevTools (onglet "Lighthouse") | [Lighthouse](https://developer.chrome.com/docs/lighthouse/overview/) |
| **WAVE** | Extension navigateur | [WAVE](https://wave.webaim.org/) |
| **htmlhint** | `npm install -g htmlhint` puis `htmlhint server/views/**/*.ejs` | [htmlhint](https://htmlhint.com/) |
| **NOUVEAU** | `npm run test:accessibility` | Exécute le build et vérifie. |

### Tests manuels
- **Navigation au clavier** : Tester avec `Tab`, `Shift+Tab`, `Entrée`, et `Échap`.
- **Lecteur d'écran** : Utiliser **NVDA** (Windows) ou **VoiceOver** (macOS).
- **Zoom à 200%** : Vérifier que le site reste utilisable.
- **Mode haute contraste** : Tester avec un thème haute contraste (Windows/macOS).
- **Mode gris** : Vérifier que le site reste compréhensible sans couleurs.

## 📝 Exemples de code conforme RGAA

### Images
```html
<!-- ✅ Bon : Alternatif descriptif -->
<img src="/assets/images/logo.png" alt="Logo de Cris et Ronronnements" />

<!-- ❌ À éviter : Alternatif vide ou non descriptif -->
<img src="/assets/images/logo.png" alt="" />
<img src="/assets/images/logo.png" alt="image" />
```

### Liens
```html
<!-- ✅ Bon : Texte descriptif -->
<a href="/bfk">En savoir plus sur le projet BFK</a>

<!-- ❌ À éviter : Texte non descriptif -->
<a href="/bfk">Cliquez ici</a>
<a href="/bfk">Plus d'infos</a>
```

### Formulaires
```html
<!-- ✅ Bon : Étiquette associée -->
<label for="email">Adresse e-mail</label>
<input type="email" id="email" name="email" />

<!-- ✅ Bon : aria-label si pas de label visible -->
<input type="email" name="email" aria-label="Adresse e-mail" />

<!-- ❌ À éviter : Pas d'étiquette -->
<input type="email" name="email" placeholder="E-mail" />
```

### Skip to main content
```html
<!-- ✅ Bon : Lien de navigation pour les lecteurs d'écran -->
<a href="#main-content" class="visually-hidden-focusable" style="position: absolute; left: -9999px; top: auto; width: 1px; height: 1px; overflow: hidden;">
    Aller au contenu principal
</a>

<!-- CSS pour le focus -->
<style>
    .visually-hidden-focusable:focus {
        position: static;
        width: auto;
        height: auto;
        overflow: visible;
        clip: auto;
    }
</style>
```

### Structure sémantique
```html
<!-- ✅ Bon : Structure claire -->
<header role="banner">
    <h1>Titre du site</h1>
    <nav aria-label="Menu principal">
        <ul>
            <li><a href="/">Accueil</a></li>
        </ul>
    </nav>
</header>

<main id="main-content" role="main" aria-labelledby="page-title">
    <h1 id="page-title">Titre de la page</h1>
    <section aria-labelledby="section-title">
        <h2 id="section-title">Titre de la section</h2>
        <p>Contenu...</p>
    </section>
</main>

<footer role="contentinfo">
    <p>&copy; 2025 Cris et Ronronnements</p>
</footer>
```

## 📚 Critères RGAA 4.1 par thème

### 1. Images
- [x] Chaque image a un attribut `alt` descriptif.
- [x] Les images décoratives ont un `alt=""`.
- [ ] Les images complexes ont une description longue (si nécessaire).

### 2. Couleurs
- [x] Le contraste texte/fond est ≥ 4.5:1.
- [x] Le contraste pour le texte grand (≥ 24px) est ≥ 3:1.
- [x] Les informations ne sont pas transmises uniquement par la couleur.

### 3. Structure
- [x] Le code HTML est valide.
- [x] Les balises HTML5 sont utilisées de manière sémantique.
- [x] La hiérarchie des titres (`<h1>` à `<h6>`) est respectée.

### 4. Navigation
- [x] Tous les liens ont un texte descriptif.
- [x] La navigation au clavier est possible.
- [x] L'ordre de tabulation est logique.
- [x] Un lien "Skip to main content" est présent.

### 5. Formulaires
- [ ] Chaque champ a une étiquette associée.
- [ ] Les messages d'erreur sont clairs et accessibles.
- [ ] Les formulaires sont utilisables au clavier.

### 6. Contenu
- [x] Le langage de la page est spécifié (`<html lang="fr">`).
- [x] Les abréviations ont un attribut `title` (si nécessaire).
- [ ] Les citations ont une balise `<cite>` (si applicable).

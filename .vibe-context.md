# Contexte : cris-et-ronronnements

## 🎯 Objectif principal
Site web pour Simon Vitsonnet avec 3 parties :
- **BFK** : Questionnaire et dictionnaire (BDSM/Fetish/Kink)
- **IUFMM** : Cours pour Maîtres/Maîtresses

**Objectif technique actuel** : Intégrer **EJS** pour simplifier la maintenance et les mises à jour.

## 🔧 Règles
1. **Jamais de données sensibles dans `/public/`**
2. Les pages d'erreur sont gérées par Express (via `server/errors/`)
3. Bootstrap et les assets publics restent dans `/public/assets/`
4. **EJS** : Tous les templates sont dans `server/views/`
5. **Dossier `/dev/`** : Exclu de Git (pour développement local uniquement)

## 📦 Dépendances
- `express` : Serveur web
- `ejs` : Moteur de templates
- `cookie-parser` : Gestion des cookies (contrôle parental)
- `gray-matter` + `marked` : Parsing Markdown

## 💻 Commandes
- Démarrer le serveur : `npm start`
- Vérifier l'état : `git status`
- Installer les dépendances : `npm install`

## 🎨 EJS - Bonnes pratiques
- **Inclure un partial** : `<%- include('partials/header') %>`
- **Passer des variables** : `res.render('pages/index', { title: 'Mon titre' })`
- **Layout principal** : Toujours utiliser `layouts/main.ejs` comme base
- **Chemins** : Utiliser des chemins relatifs à `/` (ex: `/assets/css/style.css`)

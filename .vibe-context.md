# Contexte du Projet : Cris et Ronronnements

## 📌 **Description Générale**
**Cris et Ronronnements** est un site web dédié à l'exploration de soi et des autres à travers des **questionnaires**, des **ressources éducatives** et des **œuvres littéraires**. Le projet aborde des thèmes comme le **BDSM**, le **Fetish**, le **Kink**, et propose des outils pour une introspection structurée et respectueuse.

---

## 🎯 **Objectifs Principaux**
1. **Projet BFK** : Questionnaire complet pour explorer ses préférences dans les domaines du BDSM, Fetish et Kink.
2. **IUFMM** : Institut Utile de Formation des Maîtres et Maîtresses, proposant des cours et ressources pédagogiques.
3. **Éditions SV** : Publication d’œuvres littéraires (poèmes, recueils) de Simon Vitsonnet.
4. **Accessibilité et Inclusivité** : Respect des normes **RGAA** (Référentiel Général d’Amélioration de l’Accessibilité) pour un site accessible à tous.

---

## 🏗️ **Architecture Technique**

### **Backend**
- **Framework** : [Express.js](https://expressjs.com/) (v4.22.2)
- **Moteur de templates** : [EJS](https://ejs.co/) (Embedded JavaScript)
- **Langage** : Node.js (>= 20.0.0)
- **Gestion des variables d'environnement** : `dotenv-safe` (validation via `.env.example`)

### **Frontend**
- **Framework CSS** : [Bootstrap 5](https://getbootstrap.com/) (local, minifié)
- **CSS Custom** : Variables CSS pour la cohérence des couleurs (conformes RGAA)
- **JavaScript** : Vanilla JS (minifié avec Terser)
- **Preprocessing CSS** : [PostCSS](https://postcss.org/) avec:
  - `autoprefixer` (compatibilité navigateurs)
  - `cssnano` (minification et optimisation)

### **Sécurité**
- **Helmet** : En-têtes HTTP sécurisés (CSP, HSTS, XSS Protection, etc.)
- **CSP (Content Security Policy)** : Avec **nonce** pour les scripts inline
- **Rate Limiting** : 100 requêtes/15 min par IP (`express-rate-limit`)
- **Compression HTTP** : Réduction de la taille des réponses (`compression`)
- **Désactivation de `X-Powered-By`** : Pour éviter la fuite d’informations
- **Validation des requêtes POST** : Limite de taille du body (10 Ko) et nombre de paramètres (100)

### **Performance**
- **Cache long** : 1 an pour les assets statiques (`max-age=31536000, immutable`)
- **ETag et Last-Modified** : Pour une gestion optimale du cache
- **Preload des ressources critiques** : CSS et JS essentiels
- **Minification** : CSS (PostCSS) et JS (Terser)
- **Compression Gzip** : Pour toutes les réponses HTTP

### **SEO**
- **Balises méta** : Description, keywords, author
- **Open Graph** : Pour le partage sur les réseaux sociaux
- **Twitter Cards** : Pour un affichage optimisé sur Twitter
- **Canonical URLs** : Éviter le contenu dupliqué
- **Sitemap XML** : Généré dynamiquement (à implémenter)
- **robots.txt** : Configuration des crawlers

### **Accessibilité (RGAA)**
- **Balises ARIA** : `role`, `aria-label`, `aria-hidden`
- **Skip-to-content** : Lien pour accéder directement au contenu principal
- **Contraste des couleurs** : Vérifié (4.5:1 minimum)
- **Focus visible** : Styles pour les éléments focusables
- **Texte alternatif** : Pour les images (à compléter)
- **Structure sémantique** : Utilisation de `header`, `main`, `footer`, etc.

## 🚀 **Déploiement**

### **Environnement de Production (PlanetHoster)**
- **Serveur** : Passenger (gère automatiquement le port et HTTPS)
- **Node.js** : >= 20.0.0
- **Port** : Géré par Passenger (pas besoin de spécifier dans `.env`)
- **HTTPS** : Géré par Passenger (certificat SSL automatique)

### **Étapes de Déploiement**
1. **Construire les assets** :
   ```bash
   npm run predeploy
   ```
2. **Vérifier les variables d’environnement** :
   - `PUBLIC_PATH` : Chemin vers les fichiers statiques (ex: `/home/crisfahs/public_html`)
   - `VIEWS_PATH` : Chemin vers les vues (ex: `/home/crisfahs/server/views`)
   - `NODE_ENV` : `production`
3. **Redémarrer l’application** :
   - PlanetHoster gère automatiquement le redémarrage via Passenger.

### **Fichier `.env` en Production**
```env
# Environnement
NODE_ENV=production

# Chemins des fichiers (à adapter selon la structure de PlanetHoster)
PUBLIC_PATH=/home/crisfahs/public_html
VIEWS_PATH=/home/crisfahs/server/views

# Port (géré par Passenger, pas besoin de le spécifier)
# PORT=3000
```

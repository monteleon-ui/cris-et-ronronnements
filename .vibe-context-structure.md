## 🏗️ Décisions architecturales (mise à jour)

### Gestion des visiteurs sans JavaScript
Une balise `<noscript>` est incluse dans `main.ejs` pour rediriger automatiquement vers `https://simonvitsonnet.com/cris-et-ronronnemets.html`.
Le dossier `noscript/` a été supprimé car redondant avec cette solution.

### Sécurité
- Utilisation de `helmet` pour les en-têtes de sécurité HTTP.
- **Nouveau** : Utilisation de `compression` pour réduire la taille des réponses.
- **Nouveau** : CSP (Content Security Policy) avec restrictions strictes.
- **Nouveau** : HSTS (HTTP Strict Transport Security) activé.
- Utilisation de `morgan` pour le logging des requêtes.

### RGAA
Le site doit être conforme au **RGAA (Niveau AA)**. Vérifier avant chaque déploiement.

### Bootstrap
- **Changement majeur** : Passage de **Bootstrap local (8.4M)** à **Bootstrap CDN (jsdelivr)**.
- **Avantages** :
  - Réduction de 8.4M de la taille du dépôt.
  - Meilleure performance (cache partagé entre les sites).
  - Mises à jour automatiques via CDN.
- **Intégrité** : Utilisation de l'attribut `integrity` pour vérifier l'intégrité des fichiers.

### Système de Build
- **PostCSS** : Minification et autoprefixing du CSS.
- **Terser** : Minification du JavaScript.
- **Scripts npm** : Automatisation via `npm run build`.

### `app.js` (points critiques - mise à jour)
```javascript
// 1. Configuration des variables d'environnement
require('dotenv').config();

// 2. Configurer EJS
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'server/views'));

// 3. Middleware de sécurité et performance
app.use(compression()); // NOUVEAU: Compression HTTP
app.use(helmet({
  contentSecurityPolicy: { // NOUVEAU: CSP renforcé
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", "'unsafe-inline'", "cdn.jsdelivr.net"],
      styleSrc: ["'self'", "'unsafe-inline'", "cdn.jsdelivr.net"],
      imgSrc: ["'self'", "data:", "https:"]
    }
  },
  hsts: { // NOUVEAU: HSTS
    maxAge: 31536000,
    includeSubDomains: true,
    preload: true
  }
}));
app.use(morgan('dev'));

// 4. Middleware pour parser le body des requêtes POST
app.use(express.json());
app.use(express.urlencoded({ extended: true }));

// 5. Servir les fichiers statiques avec cache
app.use(express.static(PUBLIC_PATH, {
  maxAge: '1y', // NOUVEAU: Cache long
  setHeaders: (res, filePath) => {
    if (filePath.endsWith('.html')) {
      res.setHeader('Cache-Control', 'no-cache');
    }
  }
}));

// 6. Démarrage du serveur (PORT obligatoire pour Passenger)
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => { 
  console.log(`Server running on port ${PORT}`); 
});
```

# Contexte : Structure du projet cris-et-ronronnements

## 📁 Structure finale validée (PlanetHoster + Passenger)
```
nodevenv/
└──── nocstorage/               # ✅ Dossier racine pour Node.js (imposé par PlanetHoster)
    ├── app.js                # Point d'entrée Express (PORT = process.env.PORT)
    ├── package.json          # Dépendances (main: "app.js")
    ├── package-lock.json
    ├── node_modules/         # Dépendances npm
    ├── server/               # Non accessible au client
    │   └── views/
    │       ├── layouts/main.ejs      # Layout principal (inclut la balise <noscript>)
    │       ├── partials/             # Header, footer, index-content
    │       │   ├── header.ejs
    │       │   ├── footer.ejs
    │       │   └── index-content.ejs
    │       ├── pages/                # Pages EJS
    │       │   └── index.ejs
    │       └── errors/               # Pages d'erreur
    │           ├── 404.ejs
    │           └── 500.ejs
    └── public/                # Accessible au client
        └── assets/                  # CSS, JS, images, Bootstrap
            ├── Bootstrap/
            ├── css/
            ├── js/
            └── images/
public_html/                  # Dossier racine web (pour Passenger)
└── .htaccess                 # Configuration Passenger (créé manuellement sur PlanetHoster)
```

## 🔧 Décisions architecturales
- **Gestion des visiteurs sans JavaScript** : 
  Une balise `<noscript>` est incluse dans `main.ejs` pour rediriger automatiquement vers `https://simonvitsonnet.com/cris-et-ronronnemets.html`. 
  Le dossier `noscript/` a été supprimé car redondant avec cette solution.

- **Sécurité** : 
  Utilisation de `helmet` pour les en-têtes de sécurité HTTP et de `morgan` pour le logging des requêtes.

- **RGAA** : 
  Le site doit être conforme au RGAA (Niveau AA). Vérifier avant chaque déploiement.

### `app.js` (points critiques)
```javascript
// 1. Configurer EJS
app.set('view engine', 'ejs');
app.set('views', path.join(__dirname, 'server/views'));

// 2. Middleware de sécurité et logging
app.use(helmet());
app.use(morgan('dev'));

// 3. Routes
app.get('/', (req, res) => { 
    res.render('pages/index', { 
        title: 'Accueil - Cris et Ronronnements' 
    }); 
});

// 4. Démarrage du serveur (PORT obligatoire pour Passenger)
const PORT = process.env.PORT || 3000;
app.listen(PORT, () => { 
    console.log(`Server running on port ${PORT}`); 
});
```

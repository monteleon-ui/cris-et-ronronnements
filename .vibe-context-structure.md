## 🏗️ **Détail des Composants Clés**

### **1. `app.js` : Configuration Centrale**
**Rôle** : Point d'entrée de l'application Express. Configure :
- Les **middlewares** (sécurité, logging, parsing, etc.)
- Les **routes**
- Les **variables globales** (nonce, année actuelle)
- Le **démarrage du serveur**

**Middlewares dans l'ordre** :
1. `express.json()` : Parsing du body JSON
2. `express.urlencoded()` : Parsing des formulaires (limité à 10 Ko)
3. `compression()` : Compression Gzip des réponses
4. `rateLimit()` : Limitation à 100 requêtes/15 min par IP
5. `generateNonce()` : Génération d'un nonce pour le CSP
6. `currentYear()` : Ajout de l'année actuelle aux locals
7. `helmet()` : Sécurité des en-têtes HTTP (CSP, HSTS, etc.)
8. `morgan()` : Logging des requêtes (fichier + console)
9. `express.static()` : Service des fichiers statiques (avec cache)

**Variables globales** :
- `res.locals.nonce` : Nonce pour le CSP (généré à chaque requête)
- `res.locals.currentYear` : Année actuelle (pour le footer)

**Routes implémentées** :

### **3. `server/middlewares/errors.js` : Gestion des Erreurs**
**Rôle** : Middlewares pour les erreurs 404 et 500.

**Fonctions** :
- `notFound(req, res, next)` :
  - Affiche la page `errors/404.ejs` avec un statut 404.
  - Gère les erreurs de rendu (affiche un message brut si nécessaire).
- `serverError(err, req, res, next)` :
  - Log l'erreur dans la console.
  - Affiche la page `errors/500.ejs` avec un statut 500.
  - Gère les erreurs de rendu.

**Améliorations apportées** :
- Utilisation du **layout `main.ejs`** pour les pages d'erreur (au lieu de dupliquer le HTML).
- Passage de `currentYear` et `nonce` aux templates.
- Logging des erreurs de rendu.

### **4. `server/views/layouts/main.ejs` : Layout Principal**
**Rôle** : Template de base pour toutes les pages. Contient :

- Le **head** avec :
  - Balises méta (charset, viewport, description, etc.)
  - Open Graph et Twitter Cards

  - Lien vers Bootstrap CSS (local)

  - Message noscript
- Le **body** avec :
  - Skip-to-content (lien vers `#main-content`)
  - En-tête (`<%- include('../partials/header') %>`)
  - Contenu principal (`<%- body %>`)
  - Pied de page (`<%- include('../partials/footer') %>`)
  - Scripts Bootstrap JS et `config.min.js`

**Variables attendues** :
- `title` : Titre de la page
- `metaDescription` : Description pour le SEO
- `metaKeywords` : Mots-clés pour le SEO
- `canonicalUrl` : URL canonique
- `body` : Contenu HTML à insérer dans `<main>`
- `nonce` : Nonce pour le CSP (utilisé dans les balises `<script>` et `<style>` inline)

## 🔒 **Sécurité**

### **Middlewares de Sécurité**

| **Middleware** | **Package** | **Configuration** | **Rôle** |
|----------------|------------|-------------------|----------|
| Helmet | `helmet` | CSP avec nonce, HSTS, etc. | Protection contre les vulnérabilités web |
| Rate Limiting | `express-rate-limit` | 100 requêtes/15 min par IP | Protection contre les attaques par force brute |
| Compression | `compression` | Gzip | Réduction de la taille des réponses |
| Body Parser | `express.json`, `express.urlencoded` | Limite à 10 Ko, 100 paramètres | Protection contre les payloads volumineux |
| X-Powered-By | Désactivé | `app.disable('x-powered-by')` | Évite la fuite d’informations |

### **Content Security Policy (CSP)**
**Configuration dans `app.js`** :
```javascript
helmet({
  contentSecurityPolicy: {
    directives: {
      defaultSrc: ["'self'"],
      scriptSrc: ["'self'", (req, res) => `'nonce-${res.locals.nonce}'`],
      styleSrc: ["'self'", "'unsafe-inline'"],
      imgSrc: ["'self'", "data:", "https:"],
      fontSrc: ["'self'"],
      connectSrc: ["'self'"],
      frameSrc: ["'none'"],
      objectSrc: ["'none'"],
      baseUri: ["'self'"],
      formAction: ["'self'"]
    }
  }
})
```

**Explications** :
- `defaultSrc: ['self']` : Par défaut, seul le site lui-même est autorisé.
- `scriptSrc` : Autorise les scripts du site + ceux avec le **nonce** (pour les scripts inline).
- `styleSrc` : Autorise les styles du site + `unsafe-inline` (nécessaire pour Bootstrap et les styles inline).
- `imgSrc` : Autorise les images du site, les data URIs et HTTPS.
- `frameSrc: ['none']` : Bloque tous les iframes (sécurité).
- `objectSrc: ['none']` : Bloque les objets (Flash, etc.).

### **Gestion des Erreurs**
- **404** : Page personnalisée (`errors/404.ejs`) avec lien vers l'accueil.
- **500** : Page personnalisée (`errors/500.ejs`) avec bouton de rechargement.
- **Logging** : Les erreurs sont loguées dans `logs/access.log` (via Morgan) et la console.

## ♿ **Accessibilité (RGAA)**

### **Mesures Implémentées**

#### **Structure HTML**
- **DOCTYPE HTML5** : `<!DOCTYPE html>`
- **Langue** : `<html lang="fr">`
- **Structure sémantique** :
  - `<header role="banner">` : En-tête du site
  - `<main id="main-content" role="main">` : Contenu principal
  - `<footer role="contentinfo">` : Pied de page
  - `<nav role="navigation" aria-label="Menu principal">` : Navigation

#### **Navigation**
- **Skip-to-content** : Lien en haut de page pour accéder directement au contenu principal.
  ```html
  <a href="#main-content" class="visually-hidden-focusable" style="...">Aller au contenu principal</a>
  ```
- **Focus visible** : Styles pour les éléments focusables (`.visually-hidden-focusable:focus`).

#### **Contraste des Couleurs**
- **Variables CSS** : Couleurs définies avec un contraste vérifié (4.5:1 minimum sur fond blanc).
  - Exemple : `--primary-color: #0056b3` (contraste 7.5:1 sur blanc).
- **Texte alternatif** : À ajouter pour toutes les images (non implémenté actuellement).

#### **Balises ARIA**
- **Rôles** : `role="banner"`, `role="main"`, `role="contentinfo"`, `role="navigation"`
- **Labels** : `aria-label` pour les liens et boutons (ex: `aria-label="Menu Projet BFK"`)
- **États** : `aria-expanded`, `aria-haspopup` pour les menus déroulants

#### **Formulaires**
- **Labels** : Chaque `<input>` doit avoir un `<label>` associé (à vérifier).
- **Messages d’erreur** : Accessibles et associés à leur champ (à implémenter si formulaires ajoutés).

### **À Améliorer**
- [ ] **Audit RGAA complet** : Utiliser [Tanaguru](https://www.tanaguru.com/) ou [AccessiNum](https://accessinum.github.io/)
- [ ] **Tests utilisateurs** : Faire tester par des personnes en situation de handicap
- [ ] **Documentation d’accessibilité** : Ajouter une page dédiée
- [ ] **Texte alternatif** : Ajouter `alt` pour toutes les images
- [ ] **Légendes pour les tableaux** : Ajouter `<caption>` si des tableaux sont utilisés

### **Mesures Implémentées**

#### **Cache**
- **Assets statiques** : Cache de 1 an (`max-age=31536000, immutable`)
  ```javascript
  app.use(express.static(PUBLIC_PATH, {
    maxAge: '1y',
    etag: true,
    lastModified: true,
    setHeaders: (res, filePath) => {
      if (filePath.match(/\.(css|js|png|jpg|jpeg|gif|ico|svg|woff2?|webp)$/)) {
        res.setHeader('Cache-Control', 'public, max-age=31536000, immutable');
      }
    }
  }));
  ```
- **HTML** : Pas de cache (`no-cache, no-store, must-revalidate`)

#### **Compression**
- **Gzip** : Activé pour toutes les réponses via `compression()`

#### **Minification**
- **CSS** : `postcss` + `cssnano` (suppression des commentaires, optimisation)
- **JS** : `terser` (compression + mangling)

#### **Preload**
- **Ressources critiques** : CSS et JS de Bootstrap et custom
  ```html
  <link rel="preload" href="/assets/bootstrap/css/bootstrap.min.css" as="style">
  <link rel="preload" href="/assets/css/style.min.css" as="style">
  <link rel="preload" href="/assets/bootstrap/js/bootstrap.bundle.min.js" as="script">
  <link rel="preload" href="/assets/js/config.min.js" as="script">
  ```

#### **Images**
- **Formats modernes** : WebP pour les icônes (`favicon.webp`)
- **Compression** : Images optimisées (à vérifier avec [TinyPNG](https://tinypng.com/))

## ⚠️ Problèmes courants et solutions (mise à jour)

| Problème | Cause | Solution |
|----------|-------|----------|
| **"It works!" s'affiche** | Passenger charge une app par défaut | Corriger `.htaccess` + supprimer `public_html/index.html` |
| `npm command not found` | Node.js/npm non activé | `source /home/.../20/bin/activate` |
| `pm2 command not found` | PM2 non installé globalement | Utiliser le chemin absolu `/home/.../node_modules/.bin/pm2` |
| **Erreur 500 en local** | Variable `body` non définie dans EJS | Vérifier que `index.ejs` passe la variable `body` à `main.ejs`. |
| **NOUVEAU** Bootstrap manquant | Bootstrap local supprimé | Utiliser le CDN (déjà configuré dans main.ejs). |
| **NOUVEAU** Fichiers minifiés manquants | Build non exécuté | Exécuter `npm run build` avant de démarrer. |
| **NOUVEAU** Erreur CSP | Script/Style bloqué | Vérifier les directives CSP dans app.js. |

## 📦 Dépendances critiques (mise à jour)

Les dépendances suivantes sont **essentielles** pour le projet :

### Production
- **`express`** : Framework web (^4.22.2).
- **`ejs`** : Moteur de templates (^3.1.10).
- **`helmet`** : Sécurité HTTP (^7.1.0) - **Ne pas supprimer**.
- **`morgan`** : Logging des requêtes (^1.10.0) - **Ne pas supprimer**.
- **`compression`** : Compression HTTP (^1.7.4) - **NOUVEAU**.
- **`dotenv`** : Variables d'environnement (^16.4.5) - **NOUVEAU**.
- **`gray-matter`** et **`marked`** : Pour le traitement du Markdown (œuvres littéraires).

### Développement
- **`nodemon`** : Rechargement automatique (^3.1.7).
- **`postcss`** + **`postcss-cli`** + **`autoprefixer`** + **`cssnano`** : Build CSS (^8.4.47, ^11.0.0, ^10.4.20, ^7.0.5).
- **`terser`** : Minification JS (^5.34.1).
- **`eslint`** : Linting (^9.15.0).

> ⚠️ **Ne pas supprimer ces dépendances sans validation préalable.**

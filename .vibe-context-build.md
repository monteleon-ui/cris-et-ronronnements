# Contexte : Système de Build Automatisé pour cris-et-ronronnements

## 🎯 Objectif du système de build
Automatiser la **minification** et l'**optimisation** des assets statiques (CSS, JavaScript) pour améliorer les performances du site.

---

## 🔧 Architecture du système de build

### Outils utilisés
| Outil | Version | Rôle |
|-------|---------|------|
| **PostCSS** | ^8.4.47 | Transformation du CSS (minification, autoprefixing) |
| **cssnano** | ^7.0.5 | Plugin PostCSS pour la minification CSS |
| **autoprefixer** | ^10.4.20 | Ajout des préfixes vendeurs pour la compatibilité |
| **Terser** | ^5.34.1 | Minification et obfuscation du JavaScript |
| **npm scripts** | - | Orchestration des tâches de build |

## 🚀 Scripts npm disponibles

### Scripts principaux
| Commande | Description | Utilisation |
|----------|-------------|-------------|
| `npm run build` | Minifie CSS et JS | Développement et production |
| `npm run build:css` | Minifie uniquement le CSS | Développement CSS |
| `npm run build:js` | Minifie uniquement le JS | Développement JS |
| `npm run build:all` | Alias de `npm run build` | - |
| `npm run clean` | Supprime les fichiers minifiés | Nettoyage |

| `npm run lint:fix` | Corrige les erreurs ESLint | Nettoyage de code |

## 📄 Configuration détaillée

### postcss.config.js
```javascript
module.exports = {
  plugins: [
    require('autoprefixer'),    // Ajoute les préfixes vendeurs
    require('cssnano')({         // Minifie le CSS
      preset: 'default',        // Utilise les présélections par défaut
    })
  ]
};
```

### package.json (scripts)
```json
{
  "scripts": {
    "build": "npm run build:css && npm run build:js",
    "build:css": "postcss public/assets/css/style.css --output public/assets/css/style.min.css --env production",
    "build:js": "terser public/assets/js/config.js --compress --mangle --output public/assets/js/config.min.js",
    "build:all": "npm run build",
    "clean": "rm -rf public/assets/css/style.min.css public/assets/js/config.min.js",
    "predeploy": "npm run clean && npm run build:all"
  }
}
```

## 📊 Optimisations appliquées par le build

### CSS (via PostCSS + cssnano)
- **Minification** : Suppression des espaces, commentaires, sauts de ligne
- **Autoprefixing** : Ajout automatique des préfixes vendeurs (`-webkit-`, `-moz-`, etc.)
- **Optimisation** : Fusion des règles similaires, simplification des sélecteurs
- **Résultat** : Réduction de ~40-60% de la taille du CSS

### JavaScript (via Terser)
- **Minification** : Suppression des espaces, commentaires, sauts de ligne
- **Obfuscation** : Raccourcissement des noms de variables (option `--mangle`)
- **Compression** : Optimisation des expressions (option `--compress`)
- **Résultat** : Réduction de ~50-70% de la taille du JS

## ⚠️ Problèmes courants et solutions

| Problème | Cause | Solution |
|----------|-------|----------|
| **Fichiers minifiés manquants** | Build non exécuté | Exécuter `npm run build` |
| **Erreurs PostCSS** | Syntaxe CSS invalide | Vérifier `style.css` avec un linter |
| **Erreurs Terser** | Syntaxe JS invalide | Vérifier `config.js` avec ESLint |
| **Fichiers source manquants** | Fichiers supprimés | Restaurer `style.css` et `config.js` |
| **Build lent** | Trop de fichiers | Optimiser le CSS/JS source |

## 🛠️ Personnalisation du build

### Ajouter un nouveau fichier CSS à minifier
1. Ajouter le fichier dans `public/assets/css/` (ex: `custom.css`)
2. Ajouter un script dans `package.json` :
   ```json
   "build:custom-css": "postcss public/assets/css/custom.css --output public/assets/css/custom.min.css --env production"
   ```
3. Mettre à jour le script `build` :
   ```json
   "build": "npm run build:css && npm run build:js && npm run build:custom-css"
   ```

### Ajouter un nouveau fichier JS à minifier
1. Ajouter le fichier dans `public/assets/js/` (ex: `app.js`)
2. Ajouter un script dans `package.json` :
   ```json
   "build:app-js": "terser public/assets/js/app.js --compress --mangle --output public/assets/js/app.min.js"
   ```
3. Mettre à jour le script `build` :
   ```json
   "build": "npm run build:css && npm run build:js && npm run build:app-js"
   ```

## 🔗 Intégration avec le layout principal

### Dans `main.ejs`
```html
<!-- CSS minifié -->
<link rel="stylesheet" href="/assets/css/style.min.css">

<!-- JS minifié -->
<script src="/assets/js/config.min.js"></script>
```

### Dans `app.js`
```javascript
// Servir les fichiers statiques avec cache long
app.use(express.static(PUBLIC_PATH, {
  maxAge: '1y', // Cache pour 1 an
  setHeaders: (res, filePath) => {
    if (filePath.endsWith('.html')) {
      res.setHeader('Cache-Control', 'no-cache');
    }
    // Cache long pour les assets minifiés
    if (filePath.match(/\.(css|js)$/)) {
      res.setHeader('Cache-Control', 'public, max-age=31536000, immutable');
    }
  }
}));
```

## 📚 Bonnes pratiques

### 1. Toujours travailler avec les fichiers source
- Modifier **uniquement** `style.css` et `config.js`
- **Ne jamais modifier** `style.min.css` ou `config.min.js` manuellement

### 2. Tester avant de commiter
```bash
# Builder les assets
npm run build

# Vérifier que le site fonctionne
npm start

# Vérifier les tailles
ls -lh public/assets/css/ public/assets/js/
```

### 3. Exclure les fichiers générés de Git
Les fichiers minifiés sont **exclu du dépôt Git** via `.gitignore` :
```gitignore
public/assets/css/style.min.css
public/assets/js/config.min.js
```

### 4. Déployer avec les assets minifiés
```bash
# Toujours exécuter le build avant déploiement
npm run predeploy
```

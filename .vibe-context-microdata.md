# Contexte Microdonnées (Schema.org) : Cris et Ronronnements

## 🎯 **Objectif**
Ce document détaille l’**implémentation des microdonnées Schema.org** sur le site **Cris et Ronronnements**. Les microdonnées permettent aux **moteurs de recherche** (Google, Bing, etc.) de mieux comprendre le contenu du site et d’afficher des **résultats enrichis** (Rich Snippets) dans les pages de résultats.

---

## 📌 **Qu’est-ce que Schema.org ?**

**Schema.org** est un **vocabulaire commun** pour structurer les données sur le Web. Il est soutenu par les principaux moteurs de recherche (Google, Bing, Yahoo, Yandex) et permet de décrire :
- **Les organisations** (ex: Cris et Ronronnements)
- **Les personnes** (ex: Simon Vitsonnet)
- **Les événements** (ex: formations, ateliers)
- **Les articles** (ex: poèmes, ressources)
- **Les produits** (ex: livres, recueils)
- **Les avis** (ex: témoignages)
- **Et bien plus**...

**Avantages** :
✅ **Meilleur référencement** (SEO) dans les moteurs de recherche.
✅ **Résultats enrichis** (Rich Snippets) : étoiles, images, descriptions détaillées.
✅ **Compréhension améliorée** du contenu par les robots d’indexation.

## 🏗️ **Implémentation Actuelle**

### **1. Microdonnées Globales (Organization + WebSite)**

**Localisation** : `server/views/layouts/main.ejs` (dans le `<head>`)

**Code** :
```html
<script type="application/ld+json" nonce="<%= nonce %>">
{
  "@context": "https://schema.org",
  "@type": "Organization",
  "name": "Cris et Ronronnements",
  "url": "https://cris-et-ronronnements.net",
  "logo": "https://cris-et-ronronnements.net/assets/images/favicon.ico",
  "description": "Site dédié à l'exploration de soi et des autres à travers des questionnaires, des ressources et des œuvres littéraires.",
  "hasPart": {
    "@type": "WebSite",
    "name": "Cris et Ronronnements",
    "url": "https://cris-et-ronronnements.net",
    "potentialAction": {
      "@type": "SearchAction",
      "target": "https://cris-et-ronronnements.net/?s={search_term_string}",
      "query-input": "required name=search_term_string"
    }
  }
}
</script>
```

**Explications** :
- **`@context`** : Définit le vocabulaire utilisé (`https://schema.org`).
- **`@type`** : Type de l’entité (`Organization` pour le site).
- **`name`** : Nom de l’organisation.
- **`url`** : URL du site.
- **`logo`** : URL du logo (favicon).
- **`description`** : Description du site.
- **`hasPart`** : Indique que l’organisation a un site web (`WebSite`).
- **`potentialAction`** : Définit une action de recherche (pour les moteurs de recherche).

**Impact SEO** :
- Google comprend que **Cris et Ronronnements** est une **organisation** avec un site web.
- Le logo peut apparaître dans les résultats de recherche.
- La description est utilisée pour les extraits.

### **3. Balises Canonical**

**Localisation** : `server/views/layouts/main.ejs` (dans le `<head>`)

**Code** :
```html
<link rel="canonical" href="https://cris-et-ronronnements.net<%= canonicalUrl || '' %>">
```

**Explications** :
- **`rel="canonical"`** : Indique aux moteurs de recherche quelle est la **version principale** d’une page.
- **Utilité** : Évite le **contenu dupliqué** (ex: `/bfk` et `/bfk/` sont considérés comme la même page).

**Impact SEO** :
- Évite la **dilution du référencement** entre des URLs similaires.
- Concentre le **juice SEO** sur une seule URL.

## 🛠️ **Implémentation Technique**

### **1. Où Placer les Microdonnées ?**

Les microdonnées **Schema.org** doivent être placées dans le `<head>` ou le `<body>` de la page, sous forme de :
- **JSON-LD** (recommandé) :
  ```html
  <script type="application/ld+json">
  { ... }
  </script>
  ```
- **Microdata** (alternative) :
  ```html
  <div itemscope itemtype="https://schema.org/Organization">
    <span itemprop="name">Cris et Ronronnements</span>
  </div>
  ```
- **RDFa** (alternative) :
  ```html
  <div vocab="https://schema.org/" typeof="Organization">
    <span property="name">Cris et Ronronnements</span>
  </div>
  ```

**Recommandation** : Utiliser **JSON-LD** (plus simple, plus lisible, recommandé par Google).

---

### **2. Comment Générer les Microdonnées ?**

#### **a. Manuellement**
- Utiliser la [documentation Schema.org](https://schema.org/docs/full.html).
- Valider avec [Google’s Rich Results Test](https://search.google.com/test/rich-results).

#### **b. Avec des Outils**
1. **[Google’s Structured Data Markup Helper](https://www.google.com/webmasters/markup-helper/)** :
   - Sélectionnez le type de données (ex: Organization, Event).
   - Entrez l’URL ou le code HTML.
   - L’outil génère le code JSON-LD.
2. **[Schema Markup Generator](https://technicalseo.com/tools/schema-markup-generator/)** :
   - Générateur avancé pour tous les types Schema.org.
3. **[JSON-LD Playground](https://json-ld.org/playground/)** :
   - Testez et validez votre JSON-LD.

#### **c. Avec des Bibliothèques**
- **Node.js** :
  ```bash
  npm install schema-org
  ```
  Exemple :
  ```javascript
  const { Organization } = require('schema-org');
  const org = new Organization({
    name: 'Cris et Ronronnements',
    url: 'https://cris-et-ronronnements.net',
    logo: 'https://cris-et-ronronnements.net/assets/images/favicon.ico',
    description: 'Site dédié à l\'exploration de soi et des autres...'
  });
  console.log(org.toJSON());
  ```

### **3. Comment Valider les Microdonnées ?**

#### **a. Google’s Rich Results Test**
- **URL** : [https://search.google.com/test/rich-results](https://search.google.com/test/rich-results)
- **Fonctionnement** :
  1. Entrez l’URL de votre page.
  2. Google analyse les microdonnées et affiche les **résultats enrichis** possibles.
  3. Signale les **erreurs** et **avertissements**.

#### **b. Schema Markup Validator**
- **URL** : [https://validator.schema.org/](https://validator.schema.org/)
- **Fonctionnement** :
  1. Collez votre code JSON-LD ou l’URL de la page.
  2. L’outil valide la syntaxe et la conformité Schema.org.

#### **c. Google Search Console**
- **URL** : [https://search.google.com/search-console](https://search.google.com/search-console)
- **Fonctionnement** :
  1. Ajoutez votre site à Search Console.
  2. Allez dans **Améliorations** > **Données structurées**.
  3. Google affiche les **erreurs** et **avertissements** pour vos microdonnées.

## 📈 **Bonnes Pratiques**

### **1. Utiliser le Bon Type Schema.org**
- **Organization** : Pour une entreprise ou une organisation.
- **Person** : Pour un individu (ex: Simon Vitsonnet).
- **WebSite** : Pour un site web.
- **WebPage** : Pour une page spécifique.
- **Article** : Pour un article de blog.
- **Course** : Pour une formation ou un cours.
- **Event** : Pour un événement.
- **Product** : Pour un produit.
- **CreativeWork** : Pour une œuvre créative (poème, livre, etc.).

### **2. Être Spécifique**
- Plus vos microdonnées sont **détaillées**, mieux c’est.
- Exemple :
  - ❌ Mauvais : `
    {
      "@type": "Thing",
      "name": "Cris et Ronronnements"
    }`
  - ✅ Bon :
    ```json
    {
      "@type": "Organization",
      "name": "Cris et Ronronnements",
      "url": "https://cris-et-ronronnements.net",
      "logo": "https://cris-et-ronronnements.net/assets/images/favicon.ico",
      "description": "Site dédié à l'exploration de soi et des autres...",
      "sameAs": [
        "https://facebook.com/crisetronronnements",
        "https://twitter.com/crisetronronnements"
      ]
    }
    ```

### **3. Valider Régulièrement**
- Utilisez [Google’s Rich Results Test](https://search.google.com/test/rich-results) après chaque mise à jour.
- Vérifiez les erreurs dans [Google Search Console](https://search.google.com/search-console).

### **4. Éviter le Spam**
- Ne pas **surcharger** une page avec trop de microdonnées.
- Ne pas utiliser de microdonnées **fausses ou trompeuses** (risque de pénalité SEO).
- Ne pas **cacher** du contenu aux utilisateurs (les microdonnées doivent refléter le contenu visible).

### **5. Mettre à Jour les Microdonnées**
- Si le contenu de votre site change (ex: nouvelle formation, nouveau livre), mettez à jour les microdonnées.
- Exemple : Si vous ajoutez une nouvelle formation, ajoutez-la à la liste `hasCourse` de IUFMM.
